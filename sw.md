Mung-AI: AI 기반 반려동물 훈련 플랫폼

핵심 기능 설명

Mung-AI 'AI 자세 분석' 

기능 작동 원리 분석

자체적으로 직접 훈련시킨 AI 모델(YOLOv8)을 사용하여, 

영상 속 반려견의 17개 주요 관절 움직임을 정밀하게 추적하고, 그 데이터를 분석하여 질병의 초기 이상 징후를 발견합니다.

작동 순서:

데이터 수집 (영상 업로드):

사용자는 앱의 안내에 따라, 반려견이 평평한 곳에서 서 있는는 모습을 스마트폰으로 촬영하여 업로드합니다.

AI 모델 준비 (백엔드 서버): * 

영상 파일은 백엔드 Python 서버로 전송됩니다. * 

서버에서는 미리 준비된, 강아지 전용 데이터셋으로 직접 

훈련시킨 YOLOv8 모델 (best.pt 파일)을 로드합니다. * 

이것이 핵심 기술입니다. 일반적인 사람용 모델이 아닌, 수많은 강아지 이미지와 관절 위치 데이터를 학습시킨 전용 모델이기 때문에, 다양한 견종과 체형에 대한 인식률과 정확도가 매우 높습니다.

관절 위치 추적 (프레임 단위 분석): * 

서버는 업로드된 영상을 1초에 수십 장의 사진(프레임)으로 나눕니다. * 그리고 각각의 프레임마다 준비된 AI 모델을 적용하여, 

아래와 같이 미리 정의된 17개 주요 관절의 (x, y) 좌표를 모두 찾아냅니다. 

 주요 관절: 코, 눈, 귀, 어깨, 팔꿈치, 손목, 엉덩이, 무릎, 발목 등

데이터 분석 (생체역학적 지표 계산): * 

수집된 모든 프레임의 관절 좌표 데이터를 바탕으로, 

시스템은 다음과 같은 핵심 생체역학 지표들을 계산합니다. * 

보행 대칭성 (Gait Symmetry): 

왼쪽 다리와 오른쪽 다리의 움직임이 얼마나 대칭적인지를 분석합니다. 한쪽에만 문제가 생기면 이 대칭이 무너지게 됩니다. * 

관절 가동범위 (Joint Angle Range): 

엉덩이, 무릎 등의 관절이 정상 범위 내에서 부드럽게 굽혀지고 펴지는지를 각도로 계산합니다. * 

결과 생성 및 시각화: 

분석이 완료되면, 백엔드는 계산된 지표들을 종합하여 '정상', '주의', '이상 징후 의심' 과 같은 최종 평가와 함께 상세 분석 리포트를 JSON 데이터로 생성합니다. * 

이 데이터는 다시 앱으로 전송되어, 사용자가 이해하기 쉽도록 다음과 같은 시각적인 형태로 제공됩니다. *

AI 훈련 추천 기능

처음 어떤 훈련을 시작해야 할지 막막할 때, AI가 즉시 맞춤형 훈련 2가지를 추천해주는 기능입니다.

데이터 수집 (사용자 입력): 

사용자는 먼저 자신의 강아지 프로필 (견종, 나이, 건강 상태 등)을 입력합니다. * 가장 중요한 '짖음 감소', '분리불안 개선' 과 같은 '핵심 훈련 목표'를 선택합니다.

AI에게 질문 준비 (프롬프트 엔지니어링): * 

시스템은 수집된 '강아지 프로필'과 '핵심 훈련 목표'를 조합하여, 

Gemini AI가 가장 잘 이해할 수 있는 상세한 명령어

(프롬프트)를 만듭니다. 

이 명령어는 단순히 정보를 나열하는 것을 넘어, 다음과 같이 AI에게 역할을 부여하고 원하는 결과물의 형식까지 지정합니다.

AI의 답변 생성:  

이 상세한 명령어를 받은 Gemini AI는, 마치 전문 훈련사처럼 사용자의 요구사항을 분석하여 맞춤형 훈련 계획 2개를 생성합니다. 

결과 표시: 

시스템은 AI로부터 받은 JSON 데이터를 앱 화면에 사용자가 보기 좋은 카드 

형태로 보여줍니다. 사용자는 이 추천안을 보고 즉시 훈련을 시작할 수 있습니다.

Part 2: 장기 계획 추천 (AI 훈련 챌린지)

일회성 추천을 넘어, 꾸준한 훈련을 통해 반려견의 성장을 유도하는 기능입니다.

데이터 분석 (훈련 기록 활용): 

사용자가 앱을 통해 훈련을 완료할 때마다, 그 결과(성공 여부, 소요 시간 등)는

 데이터베이스(training_history 테이블)에 차곡차곡 쌓입니다.

심층 분석 요청 (기록 기반 프롬프트):  

사용자가 'AI 훈련 챌린지 시작하기'를 누르면, 시스템은 저장된 모든 훈련 기록을 가져옵니다. * 

그리고 이 기록 데이터를 전부 담아 Gemini AI에게 새로운 명령을 내립니다. > 

"당신은 데이터 분석가이자 훈련 코치입니다. 이 강아지의 과거 훈련 기록 전체를 심층적으로 분석하고, 

성공률이 정체된 부분이나 발전 패턴을 파악하여, 앞으로의 성장을 위한 '14일 챌린지' 계획을 1개만 생성해주세요."

AI의 장기 플랜 생성: 

AI는 누적된 데이터를 분석하여 '이 강아지는 현재 '기다려' 훈련에 어려움을 겪고 있구나' 와 같은 핵심 성장 목표를 스스로 파악합니다. 

그 후, '기초 다지기 → 실전 마스터(11~14일차)'와 같이 체계적인 14일짜리 훈련 계획을 만들어냅니다.

훈련 상담 챗봇 기능

Mung-AI 'Gemini 챗' 기능 작동 원리 분석

사용자가 대화 상대로 느끼는 'AI 훈련 코치'는 사실, 사전에 정의된 역할(Persona)과 

사용자의 강아지 데이터를 함께 전달받은 Gemini AI입니다.

AI 코치 역할 부여 (System Prompt 설정):  

채팅 기능이 시작되면, 시스템은 가장 먼저 Supabase Edge Function (gemini-plaintext-chat)을 통해 Gemini AI에게 

명확한 역할을 부여합니다. * 코드에 따르면, AI는 다음과 같은 시스템 명령어를 받습니다. > "You are a helpful and friendly dog training coach. Respond in plain text, not JSON." > 

(당신은 친절하고 도움이 되는 강아지 훈련 코치입니다. JSON이 아닌 일반 텍스트로 답변하세요.) * 

이 명령을 통해 AI는 딱딱한 정보 제공자가 아닌, 친근한 전문가의 정체성을 갖고 대화를 시작하게 됩니다.

강아지 정보 요약 (대화의 맥락 생성): 

 사용자가 첫 메시지를 보내기 전, 시스템은 백그라운드에서 현재 로그인된 사용자의 강아지 프로필 전체를 조회합니다. * 

GeminiChatPage.tsx 컴포넌트는 이 정보를 바탕으로 AI에게 전달할 '

사전 브리핑' 자료를 동적으로 생성합니다. 이 요약 정보는 다음과 같은 형식입니다.

 [SYSTEM] 사용자의 강아지 '몽이'에 대한 프로필 요약입니다. 이 정보를 바탕으로 전문적이고 친근한 훈련사처럼 한국어로 답변해주세요.

 2 - 견종: 푸들

 3 - 나이: 3세 

4 - 훈련 목표: 짖음 감소, 산책 시 흥분 가라앉히기 

5 - 알려진 행동: '앉아', '기다려' 가능

 6 - 건강 특이사항: 슬개골 탈구 1기

사용자 질문 + AI 브리핑 자료 결합: * 사용자가 "산책할 때 너무 흥분하는데 어떡하죠?" 라고 질문을 입력합니다. * 

시스템은 이 질문을 그냥 보내는 것이 아니라, 

[2단계] 에서 만든 '사전 브리핑' 자료와 전체 대화 기록을 함께 묶어서 Gemini AI에게 전달합니다.

AI의 맥락 기반 답변 생성:

Gemini AI는 '산책 시 흥분'이라는 사용자의 질문과 더불어, '이 강아지가 슬개골 탈구 1기'라는 사실을 이미 인지하고 있습니다.

따라서 "산책 시 흥분을 가라앉히는 것도 중요하지만, 몽이는 슬개골 탈구가 있으니 격한 움직임보다는 '코 담요' 놀이로 에너지를 분산시키는 건 어떨까요?" 와 같이, 상황과 건강 상태까지 고려한 매우 개인화된 답변을 생성할 수 있게 됩니다.

대화 표시 및 기록:

AI가 생성한 답변은 다시 앱 화면에 표시되며, 이 대화 내용은 다음 질문을 위해 계속해서 전체 대화 기록에 추가됩니다. 이를 통해 AI는 방금 나눈 대화 내용을 기억하고 다음 대화를 이어갈 수 있습니다.

Gemini 챗 기능은 다음 3가지 요소를 결합하여 고도로 개인화된 경험을 제공합니다.

정체성 (Persona): "당신은 친절한 훈련 코치야."

사전 정보 (Context): "이 강아지는 이런 특징과 목표를 가지고 있어."

사용자 질문 (Query): "이런 점이 궁금해."

이 세 가지를 종합하여 답변을 생성하기 때문에, 사용자는 마치 자신의 반려견을 잘 아는 전문가와 1:1로

 상담하는 듯한 특별한 경험을 하게 되는 것입니다.

견종 백과 기능

이 기능은 데이터베이스를 가장 순수하게 활용하는, 

이름 그대로 '백과사전' 기능입니다.

작동 원리:

사용자가 '견종백과' 메뉴에 들어오면, 앱은 데이터베이스에서 모든 견종의 목록을 가져와 보여줍니다.

사용자가 특정 견종을 검색하거나 선택하면, 해당 견종의 ID를 기반으로 데이터베이스에서 모든 상세 정보를 조회합니다.

조회된 정보(역사, 성격, 특징, 사진 등)를 미리 디자인된 상세 페이지 템플릿에 맞춰 사용자에게 보여줍니다.

핵심: 사용자가 원하는 정보를 능동적으로 탐색하고 학습할 수 있도록 하는, 데이터의 순수한 조회(Read) 및 전시(Display) 기능입니다.

견종 MBTI 테스트

이 기능은 사용자의 성향과 라이프스타일을 분석하여, 가장 잘 맞는 '성격'의 견종을 찾아주는 감성적인 접근 방식입니다.

작동 원리:

MBTI 테스트: 사용자는 "혼자 있는 시간을 즐기나요?", "새로운 사람을 만나는 것을 좋아하나요?" 와 같은 10여 개의 심리테스트 질문에 답변합니다.

성향 분석: 시스템은 사용자의 답변을 미리 정의된 로직에 따라 분석하여, 활발함(E)/차분함(I), 이성적(T)/감성적(F) 등 Mung-AI만의 '펫 라이프스타일 MBTI' 유형을 도출합니다.

견종 매칭:

핵심적으로, 견종 데이터베이스의 모든 견종에게도 사전에 이 MBTI 유형이 부여되어 있습니다. (예: 보더콜리는 '열정적인 활동가', 시츄는 '느긋한 평화주의자')

시스템은 사용자의 MBTI 결과와 견종들의 MBTI 유형을 비교하여, 일치하거나 가장 유사한 견종을 "당신의 운명의 반려견!"과 같은 형태로 추천해 줍니다.

핵심: 데이터의 정량적인 속성보다는, '성격'과 '성향'이라는 키워드로 사용자와 데이터를 감성적으로 매칭(Matching)하는 기능입니다.

견종 추천 기능

작동 원리: 

1. 필터링 질문 (추천 마법사): 

사용자는 "아파트에 거주하시나요?", "털 빠짐에 얼마나 예민하신가요?", "하루에 산책은 얼마나 가능하신가요?" 등 구체적인 조건에 답변합니다. 

2. 가중치 부여 및 점수 계산: * 

이 요청은 breed-recommender 라는 Supabase Edge Function으로 전송됩니다. 

사용자가 '아파트 거주'를 선택하면, 데이터베이스의 모든 견종 중 '아파트 적합성' 점수가 높은 견종에게 높은 가중치를 부여합니다. * '훈련 용이성', '필요 운동량', '친화력' 등 모든 답변을 종합하여 각 견종별로 '종합 적합도 점수'를 계산합니다. 

3. 결과 정렬: 

모든 견종의 점수 계산이 끝나면, 가장 높은 점수를 받은 순서대로 리스트를 정렬하여 사용자에게 "당신에게 가장 잘 맞는 견종은 다음과 같습니다"라며 보여줍니다.

핵심: 사용자의 현실적인 조건(Condition)을 기반으로 데이터베이스를 필터링(Filtering)하고 정렬(Sorting)하여 최적의 해답을 찾아주는 기능입니다.

Mung-AI 견종 관련 3대 기능 통합 분석

공통 기반: 견종 데이터베이스 (The Source of Truth)

세 기능의 심장에는 Supabase에 구축된 '견종 정보 테이블'이 있습니다. 이 테이블에는 각 견종의 이름, 원산지, 크기, 성격(활동성, 친화력, 훈련 용이성 등), 평균 수명, 특징, 사진 URL 등 수십 개의 상세한 속성이 저장되어 있습니다.

세 기능은 하나의 '견종 데이터베이스' 라는 원석을, '백과' 라는 창을 통해 그대로 보여주기도 하고, 'MBTI' 라는 도구로 감성적으로 깎아내기도 하며, '추천' 이라는 정교한 필터로 걸러냅니다다

훈련 영상 추천 기능

이 기능은 Google의 YouTube Data API v3를 직접 호출하여, 

전 세계의 방대한 훈련 영상 콘텐츠를 사용자에게 선별적으로 제공하는 역할을 합니다.

언어 설정에 맞는 검색어를 동적으로 결정합니다. * 

만약 사용자가 한국어 사용자라면: 

검색어(query)는 "강아지 훈련" 또는 "dog training" 과 같이 한국 사용자에게 가장 적합한 키워드로 설정됩니다. * 

만약 영어 사용자라면: 검색어는 "dog training for beginners" 와 같이 영어권 사용자에게 더 적합한 키워드로 설정됩니다.

. YouTube API 호출 (데이터 요청): * 

시스템은 결정된 검색어와 함께, 사전에 발급받은 YouTube API 키를 사용하여 Google의 YouTube Data API v3 서버에 search.list 요청을 보냅니다. 

데이터 수신 및 파싱 (정보 가공):

YouTube API 서버는 요청에 부합하는 영상 리스트를 JSON 데이터 형식으로 응답합니다.

useTrainingVideos.ts 훅은 이 복잡한 JSON 데이터에서 앱에 필요한 핵심 정보들만 추출하고 가공합니다.

영상 고유 ID (videoId)

영상 제목 (title)

영상 설명 (description)

미리보기 이미지 URL (thumbnail)

결과 표시 (UI 렌더링): *

 전달받은 데이터를 기반으로, 사용자에게 익숙한 형태의 영상 썸네일 그리드(Grid)를 화면에 그려줍니다. *

 사용자가 특정 영상을 클릭하면, 해당 videoId를 이용해 YouTube 페이지로 이동시키거나 앱 내의

 플레이어를 통해 영상을 재생합니다.
